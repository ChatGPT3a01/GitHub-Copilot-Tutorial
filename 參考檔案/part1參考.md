🧭 實作總流程
Step 0｜事前準備（只做一次）
✅ 你需要準備：

電腦（Windows）

已安裝 VS Code

已登入 GitHub 帳號

已安裝 GitHub Copilot（若尚未，Part 2 會完整教，這裡只需能看到 Copilot 建議）

🖼️ Step 1｜截圖 1-1：GitHub Copilot 官方頁面
📍 目的

讓學生「第一眼知道 Copilot 是官方工具，不是來路不明 AI」

🔧 操作步驟

打開瀏覽器

前往
👉 https://github.com/features/copilot

畫面停在最上方

要看到：

GitHub Copilot Logo

主視覺插圖

標題文字

📸 截圖方式（Windows）

按下 Win + Shift + S

選擇「矩形截圖」

框住整個頁面上半部

👉 存檔命名建議：
part1_1_github_copilot_home.png

✅ 完成後不用跟我說，先繼續下一張

🖼️ Step 2｜截圖 1-2：Copilot 自動補全（最重要）

這一張是 整個 Part 1 的靈魂圖

🔧 操作步驟

打開 VS Code

新建檔案

檔名：demo.py

輸入以下內容（只輸入這一行）：

# 計算兩個數字的加總


停下來，不要按 Enter

等 1～2 秒
👉 你會看到「灰色的 Copilot 建議程式碼」

📸 截圖重點

要看到：

VS Code 介面

左側檔案樹（可有可無）

灰色建議程式碼

右下角 Copilot 圖示（加分）

📸 截圖

Win + Shift + S

框整個 VS Code 視窗

👉 檔名：
part1_2_copilot_autocomplete.png

🖼️ Step 3｜截圖 1-3：中文註解生成完整函式
🔧 操作步驟

在同一個 demo.py

只貼上這兩行中文註解

# 建立一個函式，輸入學生的國文、英文、數學成績
# 回傳平均分數和等第（90以上A，80以上B，70以上C，60以上D，其他F）


停下來

等 Copilot 出現「整段灰色函式」

📸 截圖重點

一定要：

看得到中文註解

看得到 Copilot 建議的完整函式

是「灰色尚未接受」的狀態

👉 檔名：
part1_3_chinese_comment_to_code.png

🧩 Step 6｜把 5 張圖「合成 1 張圖」（重點）
✅ 最簡單做法（我最推薦給老師）
方法 A：用 PowerPoint / Google Slides（穩定）

新增一張空白投影片（16:9）

標題打：

Part 1｜認識 GitHub Copilot

依序插入 5 張圖片

排列建議：

[1-1] [1-2]
[1-3]
[1-4] [1-5]


全選 → 對齊 → 平均分布

右鍵 → 另存成圖片（PNG）

👉 輸出檔名：
part1_all_in_one.png
